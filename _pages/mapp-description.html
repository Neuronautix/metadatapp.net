A next-generation platform for reliable and efficient preclinical research

At its core, MAPP solves one of the biggest bottlenecks in preclinical R&D: scattered, inconsistent, and poorly reusable metadata. Instead of data being locked in notebooks, spreadsheets, or proprietary silos, MAPP provides a unified, API-first environment where all experimental metadata—from animal models and procedures to devices, behaviors, and outcomes—are captured in a structured, standardized, and interoperable way.

Core features

Ontology‑driven metadata

Capture experiments with validated schemas (JSON‑LD + SHACL). Align with Mouse Behavior Ontology (MBO), SEND‑JSON, and lab SOPs for semantic consistency.

HCM integration

Plug in DVC, LMT, iMouse and other systems. Normalize outputs into comparable behavioral metrics with provenance and versioning.

API Platform backend

Symfony/API Platform with GraphQL & REST. Auth via Keycloak (OIDC). Postgres storage. Exports to CSV, JSON, JSON‑LD, and NWB.

React admin UI

Fast data entry with smart forms: required fields, controlled vocabularies, date pickers, and data validators.

ML‑ready data

Clean, structured datasets with clear labels and metadata context for downstream analytics, anomaly detection, and model training.

Reproducibility by default

Version control, data lineage, and export recipes ensure the same analysis yields the same results across labs and time.

Architecture at a glance

A pragmatic three‑layer model: UI for data capture and review, API for validation and governance, and storage/services for persistence and exports.

Frontend (React Admin)

  • Smart forms, validators, vocabularies
  • Projects/Studies/Datasets dashboards
  • CSV/JSON export triggers

API (Symfony / API Platform)

  • JSON‑LD/GraphQL endpoints
  • OIDC auth, RBAC, audit
  • SHACL/schema validation

Storage & Services

  • PostgreSQL for metadata
  • Object storage for files
  • Exports: CSV, JSON, JSON‑LD, NWB

Integrations

  • HCM & instruments (DVC, LMT, etc.)
  • Adapters normalize units and formats
  • Knowledge graph contexts

Data lifecycle

1) Define

Choose an ontology‑aligned template (study, cohort, procedures). Required fields + controlled vocabularies ensure consistency.

2) Capture

Enter metadata; ingest HCM outputs through adapters with unit normalization and provenance tags.

3) Validate

Server‑side SHACL validation + client hints catch issues before they pollute datasets.

4) Export

Reproducible export recipes to CSV/JSON/JSON‑LD/NWB. Each export is versioned.

Frequently Asked Questions

Is MAPP a replacement for my Electronic Lab Notebook (ELN)?

Not necessarily. MAPP is designed to integrate with your existing tools, including ELNs and LIMS. It acts as a central hub for your metadata, enriching the data you collect in your other systems, not replacing them.

What kind of data can I manage with MAPP?

MAPP is designed to manage all the metadata related to your preclinical research. This includes information about your animal subjects (e.g., species, strain, age), experimental procedures, devices used, behavioral observations, and study outcomes.

How does MAPP help with FAIR data principles?

MAPP is built from the ground up with FAIR principles in mind. It uses JSON-LD to make your metadata machine-readable and interoperable. It provides a structured way to capture data, ensuring it is findable, accessible, and reusable.

What are the export options?

You can export your data in various formats, including CSV, JSON, and JSON-LD. We are also working on supporting other formats like NWB. This flexibility allows you to use your data in other applications, for publications, or for sharing with collaborators.

Is MAPP only for large pharmaceutical companies?

No, MAPP is designed to be scalable and can be used by academic labs, CROs, and pharmaceutical companies of all sizes. Its modular design allows it to be adapted to different workflows and needs.
